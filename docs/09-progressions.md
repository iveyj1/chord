# Progressions

Version: 0.1.0  
Last updated: 2026-01-31

This document defines chord progressions as sequences of Roman-numeral tokens with functional annotations. It provides canonical templates for classical and pop usage, suitable for generation and evaluation.

---

## 1. Definition

A **progression** is an ordered sequence of harmony tokens over time.

Canonical abstract form:

```
P = [t₀, t₁, …, tₖ]
```

where each token t includes (at minimum):
- Roman numeral (degree + quality [+ optional sevenths/inversion])
- Function label (T/PD/D)

Progressions may be parameterized by:
- Key
- Harmonic rhythm
- Phrase structure

---

## 2. Functional Templates

The minimal functional skeleton is:

```
T → PD → D → T
```

Progressions are generated by instantiating this skeleton with substitutions.

---

## 3. Cadential Archetypes

### 3.1 Authentic cadence

Common realizations:
- V → I
- V7 → I
- vii° → I

Functional form:
- D → T

### 3.2 Half cadence

Common realizations:
- ii → V
- IV → V

Functional form:
- PD → D

### 3.3 Plagal cadence

Common realizations:
- IV → I

Functional form:
- PD → T

### 3.4 Deceptive cadence

Common realizations:
- V → vi
- V7 → vi

Functional form:
- D → T-substitute

---

## 4. Common Classical Progressions

These are idiomatic in common-practice tonal harmony.

### 4.1 Basic phrase cadence

```
I  | IV  V  | I
T    PD  D    T
```

### 4.2 Predominant expansion

```
I  | ii  V  | I
T    PD  D    T
```

### 4.3 Circle-of-fifths (diatonic)

```
I  | IV  vii°  iii  vi  ii  V  | I
T    PD   D?   T?   T   PD  D    T
```

Function labels for inner terms depend on context; treat as soft.

### 4.4 Secondary dominant insertion

```
I  | V/V  V  | I
T     D   D    T
```

---

## 5. Common Pop Progressions

Pop progressions are often loop-based, with weaker directed cadence.

### 5.1 Axis progression

```
I  V  vi  IV
T  D  T   PD
```

### 5.2 Variants

```
vi  IV  I  V
T   PD  T  D

I  vi  IV  V
T  T   PD  D
```

### 5.3 Minor-key pop loop (natural minor)

```
i  ♭VII  ♭VI  ♭VII
T   D?    PD    D?
```

Function is often modal/planar; treat labels as weak unless cadence occurs.

---

## 6. Expansion and Substitution Rules

These transformations preserve overall function while varying surface harmony.

### 6.1 Tonic substitutions

- I ↔ vi
- I ↔ iii (weak)

### 6.2 Predominant substitutions

- IV ↔ ii
- ii ↔ ii6 (first inversion)

### 6.3 Dominant substitutions

- V ↔ V7
- V ↔ vii° (leading-tone chord)

### 6.4 Passing/neighbor chords (functional fillers)

Use with stepwise bass motion:
- I → I6 → ii
- I → V6 → vi

---

## 7. Harmonic Rhythm and Phrase

Progression tokens are assigned durations.

Canonical representation:

```
[(t₀, dur₀), (t₁, dur₁), ...]
```

Rules of thumb:
- Cadential chords often align with strong metric positions
- Faster harmonic rhythm increases perceived motion

---

## 8. Computational Use

### 8.1 Generation

Strategy:
1. Choose phrase length (bars)
2. Choose functional skeleton
3. Instantiate with substitution rules
4. Optionally insert secondary dominants or passing chords
5. Emit RN tokens + durations

### 8.2 Evaluation

Heuristics:
- Penalize D→PD (backtracking) unless explicitly allowed
- Reward cadence closure at phrase ends
- Enforce style profile (classical vs pop) via rule weights

---

## 9. Style Profiles (Abstract)

Define style as weightings on constraints:

- Classical: stronger directed function, stricter cadence placement
- Pop: allow loops, weaker cadence requirements, more modal mixture

This enables a single generator with different parameter sets.
